## 1. 产品概览

| 维度 | 内容 |
| --- | --- |
| 产品名称 | **GreenBite**（品牌名：FreshToday-AI） |
| 产品定位 | 香港本地有机农产品 + AI 个性化菜单订阅电商 |
| 目标市场 | 香港 (HK)，大湾区高端家庭、注重健康的白领与小型 B2B 餐饮 |
| 商业模式 | B2C 订阅制 (Recurring) + B2B 按需配送 (On-Demand) + 增值服务 (ESG 报告、AI 菜单) |
| 技术栈 | Laravel 12 · MySQL 8 · Tailwind 4 · Blade · jQuery · Gemini API |

## 2. 目标用户与痛点

### 用户 A：注重健康的家庭主妇 / 主夫（30-45 岁）
- **痛点**：超市蔬果来源不透明；不知道什么菜搭配什么菜健康；每周都要花时间规划菜单。
- **渴望**：一份"已经替我想好"的每周菜单 + 可信赖的有机食材。

### 用户 B：港岛 / 九龙高净值白领（25-40 岁）
- **痛点**：996 工作节奏没时间买菜；想健身 / 减脂但不懂营养；愿意为"省事 + 健康"付费。
- **渴望**：App 内 3 分钟完成订阅，剩下的事 AI + 配送全部搞定。

### 用户 C：精品餐厅 / 咖啡店主理人（B2B）
- **痛点**：每天清晨要去多个湿货市场比价，食材品质波动大；难以向客人讲"有机溯源"故事。
- **渴望**：一份稳定供应、带溯源证书的本地有机食材清单 + 可对客展示的 ESG 报告。

## 3. JTBD (Jobs-to-be-Done) 框架

| 层级 | 描述 |
| --- | --- |
| **Functional Job（功能性）** | 当我需要采购"本周的有机食材 + 搭配菜单"时，GreenBite 让我"在 3 分钟内订好，剩下的由 AI 排单 + 本地农场直送"。 |
| **Emotional Job（情感性）** | 让我感到"我对家人 / 客人的健康负责"，并对食材来源感到安心。 |
| **Social Job（社会性）** | 让我可以向朋友 / 客户分享"我在用 AI 帮我吃得更健康 / 我的餐厅有碳足迹报告"，塑造有品位的健康生活者形象。 |

**核心 JTBD 句式**：
> *When* 我发现"周中没时间买菜 + 想吃得健康"，  
> *I want to* 让 AI 帮我生成个性化菜单并订阅本地有机食材，  
> *So I can* 每周省下 3-5 小时，并确信家人 / 客人吃到的是真有机。

## 4. 价值主张 (Value Proposition)

> **一句话**：让香港家庭和精品餐厅，用"3 分钟订阅 + AI 菜单"吃上一周真有机本地食材，并对每份订单的碳足迹了然于心。

### 三个支撑点
1. **真溯源真有机**：直接对接香港本地有机农场，每份食材带产地 / 农户 / 碳排数据。
2. **AI 个性化菜单**：基于 Gemini 的问卷 + 历史订单 + 营养目标，生成"今天该吃什么"的周菜单。
3. **ESG 可视化**：每个订阅账户可生成可分享的"年度 ESG 报告"（家庭版 + B2B 餐厅版），差异化于纯电商竞品。

## 5. 核心指标 (North Star + Supporting Metrics)

| 层级 | 指标 | 目标 (Sprint 12 末) |
| --- | --- | --- |
| **North Star (北极星)** | 周活跃订阅订单数 (WAO - Weekly Active Orders) | ≥ 500 单 / 周 |

## 6. 范围边界 (Scope)

### ✅ In-Scope (MVP 必须交付)
- 账户与会员 (Auth / Profile / Address)
- 商品目录与有机溯源展示
- 购物车 + 单次订单
- 支付集成 (Stripe / PayMe / FPS)
- 周期性配送订阅 (Weekly / Bi-weekly)
- AI 问卷 + Gemini 菜单生成
- 碳足迹计算引擎 (基础公式)
- 运营后台 (订单 / 商品 / 用户 / 农场)

### ❌ Out-of-Scope (本期不做)
- 移动原生 App (iOS / Android) — 仅做响应式 Web
- 自营冷链物流 — 第三方配送
- 跨境配送 / 海外直采
- 直播带货 / 短视频内容电商
- 复杂的 ERP 财务对账 — 用 Stripe Dashboard
- 智能硬件 (智能冰箱 / IoT 食材识别)
